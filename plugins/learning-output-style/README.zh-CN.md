# 学习风格插件

此插件将未发布的 Learning 输出风格与解释功能结合为 SessionStart 钩子。

**注意：** 此插件与原始未发布的 Learning 输出风格不同，它还包含了 [explanatory-output-style 插件](https://github.com/anthropics/claude-code/tree/main/plugins/explanatory-output-style)的所有功能，提供交互式学习和教育性见解。

警告：除非您愿意承担此插件额外指令的 token 成本和学习模式的交互性质，否则不要安装此插件。

## 它做什么

启用后，此插件在每个会话开始时自动添加指令，鼓励 Claude：

1. **学习模式：** 通过在决策点请求有意义的代码贡献让您参与主动学习
2. **解释模式：** 提供关于实现选择和代码库模式的教育性见解

而不是自动实现所有内容，Claude 将：

1. 识别您可以编写 5-10 行有意义代码的机会
2. 专注于您的输入真正重要的业务逻辑和设计选择
3. 准备您贡献的上下文和位置
4. 解释权衡并指导您的实现
5. 在编写代码前后提供教育性见解

## 它如何工作

插件使用 SessionStart 钩子向每个会话注入额外上下文。此上下文指示 Claude 采用交互式教学方法，您主动参与编写代码的关键部分。

## Claude 何时请求贡献

Claude 将要求您编写代码：
- 带有多种有效方法的业务逻辑
- 错误处理策略
- 算法实现选择
- 数据结构决策
- 用户体验决策
- 设计模式和架构选择

## Claude 不会请求贡献

Claude 将直接实现：
- 样板或重复代码
- 没有意义选择的明显实现
- 配置或设置代码
- 简单 CRUD 操作

## 示例交互

**Claude：** 我已设置认证中间件。会话超时行为是安全与用户体验的权衡 - 会话应该在活动时自动延长，还是有硬超时？

在 `auth/middleware.ts` 中，实现 `handleSessionTimeout()` 函数以定义超时行为。

考虑：自动延长改善用户体验但可能让会话开放更长；硬超时更安全但可能让活跃用户沮丧。

**您：** [编写 5-10 行实现您偏好的方法]

## 教育性见解

除了交互式学习，Claude 将使用此格式提供关于实现选择的教育性见解：

```
`★ 见解 ─────────────────────────────────────`
[关于代码库或实现的 2-3 个关键教育要点]
`─────────────────────────────────────────────────`
```

这些见解专注于：
- 针对您代码库的具体实现选择
- 您代码中的模式和约定
- 权衡和设计决策
- 代码库特定的细节而非通用编程概念

## 用法

安装后，插件在每个会话开始时自动激活。不需要额外配置。

## 从输出风格迁移

此插件将未发布的 "Learning" 输出风格与已弃用的 "Explanatory" 输出风格结合。它提供交互式学习体验，您在有意义的决策点主动贡献代码，同时接收关于实现选择的教育性见解。

如果您之前使用 explanatory-output-style 插件，此学习插件包含所有该功能加上交互式学习功能。

此 SessionStart 钩子模式大致相当于 CLAUDE.md，但它更灵活并允许通过插件分发。

## 管理变更

- 禁用插件 - 保持代码安装在您的设备上
- 卸载插件 - 从您的设备删除代码
- 更新插件 - 创建此插件的本地副本以个性化它
  - 提示：请 Claude 阅读 https://docs.claude.com/en/docs/claude-code/plugins.md 并为您设置！

## 理念

通过做来学比被动观察更有效。此插件将您与 Claude 的交互从 "看并学" 转变为 "构建并理解"，确保您通过亲手编写有意义逻辑来发展实用技能。
