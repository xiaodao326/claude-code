# 解释性输出风格插件

此插件将已弃用的 Explanatory 输出风格 recreated 为 SessionStart 钩子。

警告：除非您愿意承担此插件额外指令和输出的 token 成本，否则不要安装此插件。

## 它做什么

启用后，此插件在每个会话开始时自动添加指令，鼓励 Claude：

1. 提供关于实现选择的教育性见解
2. 解释代码库模式和决策
3. 在任务完成与学习机会之间取得平衡

## 它如何工作

插件使用 SessionStart 钩子向每个会话注入额外上下文。此上下文指示 Claude 在编写代码前后提供简短的教育性解释，格式为：

```
`★ 见解 ─────────────────────────────────────`
[2-3 个关键教育要点]
`─────────────────────────────────────────────────`
```

## 用法

安装后，插件在每个会话开始时自动激活。不需要额外配置。

见解专注于：

- 针对您代码库的具体实现选择
- 您代码中的模式和约定
- 权衡和设计决策
- 代码库特定的细节而非通用编程概念

## 从输出风格迁移

此插件替代已弃用的 "Explanatory" 输出风格设置。如果您之前使用：

```json
{
  "outputStyle": "Explanatory"
}
```

现在可以通过安装此插件实现相同行为。

更一般地说，此 SessionStart 钩子模式大致相当于 CLAUDE.md，但它更灵活并允许通过插件分发。

注意：涉及软件开发以外任务的输出风格，最好表达为 [子代理](https://docs.claude.com/en/docs/claude-code/sub-agents)，而非 SessionStart 钩子。子代理改变系统提示，而 SessionStart 钩子添加到默认系统提示。

## 管理变更

- 禁用插件 - 保持代码安装在您的设备上
- 卸载插件 - 从您的设备删除代码
- 更新插件 - 创建此插件的本地副本以个性化此插件
  - 提示：请 Claude 阅读 https://docs.claude.com/en/docs/claude-code/plugins.md 并为您设置！
