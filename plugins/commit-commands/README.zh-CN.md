# 提交命令插件

使用简单的命令简化您的 git 工作流程，用于提交、推送和创建拉取请求。

## 概述

提交命令插件自动化常见的 git 操作，减少上下文切换和手动命令执行。不再需要运行多个 git 命令，使用单个斜杠命令处理整个工作流程。

## 命令

### `/commit`

根据已暂存和未暂存的变更自动生成提交消息，创建 git 提交。

**功能：**
1. 分析当前 git 状态
2. 审查已暂存和未暂存的变更
3. 检查最近的提交消息以匹配您仓库的风格
4. 起草适当的提交消息
5. 暂存相关文件
6. 创建提交

**用法：**
```bash
/commit
```

**示例工作流程：**
```bash
# 对代码进行一些更改
# 然后简单地运行：
/commit

# Claude 将：
# - 审查您的变更
# - 暂存文件
# - 创建带有适当消息的提交
# - 显示提交状态
```

**功能：**
- 自动起草匹配您仓库风格的提交消息
- 遵循常规提交实践
- 避免提交包含密钥的文件（.env、credentials.json）
- 在提交消息中包含 Claude Code 签名

### `/commit-push-pr`

完整工作流程命令，一步完成提交、推送和创建拉取请求。

**功能：**
1. 创建新分支（如果当前在 main）
2. 使用适当消息暂存和提交变更
3. 推送分支到 origin
4. 使用 `gh pr create` 创建拉取请求
5. 提供 PR URL

**用法：**
```bash
/commit-push-pr
```

**示例工作流程：**
```bash
# 进行您的变更
# 然后运行：
/commit-push-pr

# Claude 将：
# - 创建功能分支（如需要）
# - 提交您的变更
# - 推送到远程
# - 打开带有摘要和测试计划的 PR
# - 给您 PR URL 进行审查
```

**功能：**
- 分析分支中的所有提交（不仅是最新的）
- 创建综合 PR 描述，包含：
  - 变更摘要（1-3 个要点）
  - 测试计划清单
  - Claude Code 签名
- 自动处理分支创建
- 使用 GitHub CLI (`gh`) 创建 PR

**要求：**
- 必须安装并认证 GitHub CLI (`gh`)
- 仓库必须有名为 `origin` 的远程

### `/clean_gone`

清理已从远程仓库删除的本地分支。

**功能：**
1. 列出所有本地分支以识别 [gone] 状态
2. 识别并删除与 [gone] 分支关联的工作树
3. 删除所有标记为 [gone] 的分支
4. 提供已删除分支的反馈

**用法：**
```bash
/clean_gone
```

**示例工作流程：**
```bash
# 在 PR 合并并远程分支删除后
/clean_gone

# Claude 将：
# - 查找所有标记为 [gone] 的分支
# - 删除任何关联的工作树
# - 删除过时的本地分支
# - 报告清理了什么
```

**功能：**
- 处理常规分支和工作树分支
- 删除分支前安全删除工作树
- 显示清晰的关于已删除内容的反馈
- 如果不需要清理则报告

**何时使用：**
- 合并并删除远程分支后
- 本地分支列表充斥着过时分支时
- 定期仓库维护期间

## 安装

此插件包含在 Claude Code 仓库中。使用 Claude Code 时命令自动可用。

## 最佳实践

### 使用 `/commit`
- 提交前审查已暂存的变更
- 让 Claude 分析您的变更并匹配您仓库的提交风格
- 信任自动化消息，但验证其准确性
- 在开发期间用于常规提交

### 使用 `/commit-push-pr`
- 准备创建 PR 时使用
- 确保所有变更已完成并测试
- Claude 将分析完整分支历史用于 PR 描述
- 审查 PR 描述并在需要时编辑
- 希望最小化上下文切换时使用

### 使用 `/clean_gone`
- 定期运行以保持分支列表清洁
- 合并多个 PR 后特别有用
- 安全运行 - 仅删除已从远程删除的分支
- 帮助维护整洁的本地仓库

## 工作流程集成

### 快速提交工作流程：
```bash
# 编写代码
/commit
# 继续开发
```

### 功能分支工作流程：
```bash
# 跨多个提交开发功能
/commit  # 第一次提交
# 更多变更
/commit  # 第二次提交
# 准备创建 PR
/commit-push-pr
```

### 维护工作流程：
```bash
# 在多个 PR 合并后
/clean_gone
# 清理的工作空间准备好用于下一个功能
```

## 要求

- Git 必须安装并配置
- 对于 `/commit-push-pr`：GitHub CLI (`gh`) 必须安装并认证
- 仓库必须是带有远程的 git 仓库

## 故障排除

### `/commit` 创建空提交

**问题**：没有变更可提交

**解决方案**：
- 确保您有未暂存或已暂存的变更
- 运行 `git status` 验证变更存在

### `/commit-push-pr` 无法创建 PR

**问题**：`gh pr create` 命令失败

**解决方案**：
- 安装 GitHub CLI：`brew install gh`（macOS）或参阅 [GitHub CLI 安装](https://cli.github.com/)
- 认证：`gh auth login`
- 确保仓库有 GitHub 远程

### `/clean_gone` 未找到分支

**问题**：没有分支标记为 [gone]

**解决方案**：
- 运行 `git fetch --prune` 更新远程跟踪
- 分支必须从远程删除才会显示为 [gone]

## 提示

- **与其他工具结合**：开发期间使用 `/commit`，准备好时使用 `/commit-push-pr`
- **让 Claude 起草消息**：提交消息分析学习您仓库的风格
- **定期清理**：每周运行 `/clean_gone` 维护清洁的分支列表
- **推送前审查**：推送前始终审查提交消息和变更
